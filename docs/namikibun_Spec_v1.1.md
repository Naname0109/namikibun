# 波きぶん（namikibun）アプリ仕様書・画面設計書

**Version 1.1 | 2026年3月10日**

1日の気分の浮き沈みを、波形グラフで可視化する気分記録アプリ

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | 波きぶん（namikibun） |
| コンセプト | 1日の気分の浮き沈みを波形グラフで可視化する気分記録アプリ |
| ターゲット | 気分の浮き沈みを把握したい人（シフトワーカー、育児中の親、受験生等） |
| プラットフォーム | iOS / Android（Flutter） |
| バックエンド | なし（完全ローカル動作） |
| 開発期間 | 7日 |
| 収益モデル | 基本無料（広告表示）＋ 広告除去の買い切り課金 |

---

## 2. 競合分析・差別化戦略

### 2.1 競合アプリの不満点

主要競合「毎日まめ」のレビューから抽出した主な不満：

- 1日に1回しか記録できない（朝と夜で気分が違うのに）
- 気分の変動・波が見えない
- 気分の原因をメモする機能が弱い
- サブスク課金への不満

### 2.2 差別化ポイント

| 機能 | 毎日まめ | 波きぶん |
|------|----------|----------|
| 記録回数 | 1日 1回 | 1日 何回でもOK |
| 表示形式 | カレンダーに丸アイコン | 波形グラフ |
| 訴求 | 今日の気分を残す | 気分の浮き沈みを知る |
| 課金 | サブスク月165円〜 | 基本無料＋広告除去買い切り |
| データ保存 | クラウド同期 | 完全ローカル |

## 3. 機能仕様

### 3.1 複数回気分記録（コア機能）

デフォルトで「朝・昼・夜」の3スロット。ユーザーが自由にスロットを追加・削除・名前変更可能。

| 項目 | 詳細 |
|------|------|
| 気分入力 | 5段階（とても良い / 良い / 普通 / 悪い / とても悪い）をワンタップで選択 |
| 絵文字選択 | 各段階に対応する絵文字アイコンを表示（✨😊😐😔😢） |
| メモ | 任意のひとことメモ（最大50文字） |
| タグ | プリセットタグから選択（仕事 / 運動 / 食事 / 人間関係 / 天気 / その他） |
| スロット初期値 | 朝（6:00-12:00）/ 昼（12:00-18:00）/ 夜（18:00-24:00） |
| スロットカスタマイズ | ユーザーが名前・時間帯を自由に編集可能（例：「仕事後」「帰宅後」） |

### 3.2 記録の編集・削除

- スロットカードを長押しで編集/削除メニューを表示
- 編集：気分レベル・メモ・タグを再入力可能
- 削除：確認ダイアログの後に削除実行
- 編集時はupdated_atを更新

### 3.3 波形グラフ表示

その日の気分をなめらかな曲線（スプライン曲線）で描画。横軸：時間帯、縦軸：気分レベル。

- デイリービュー：1日の波形を大きく表示（描画アニメーション付き）
- カレンダービュー：各日にミニ波形アイコンを表示（CustomPainterで自前描画）
- グラフの色：気分レベルに応じてグラデーション（青→緑→黄→オレンジ→赤）

**データポイント数による表示ルール：**

| データポイント数 | 表示方法 |
|-----------------|---------|
| 0点（未記録） | グレーの点線 |
| 1点 | 点のみ表示 |
| 2点 | 直線で接続 |
| 3点以上 | スプライン曲線で描画 |

### 3.4 カレンダー表示

月単位のカレンダーで、各日にその日の波形ミニグラフを表示。タップするとその日の詳細に遷移。

- 各日のセルにミニ波形（CustomPainterで描画、fl_chartは使用しない）
- 未記録の日はグレーの点線で表示
- 連続記録日数をカレンダー上部に表示
- 左右スワイプで月移動

### 3.5 振り返り統計（v1.0範囲）

月単位で以下の統計を表示：

- 時間帯別の平均気分（棒グラフ）
- 月全体の気分推移（折れ線グラフ）
- 「今月のハイライト」カード（最高/最低の日とメモ）

※レーダーチャート（タグ別分析）はv1.1で追加予定。

### 3.6 リマインダー通知

ローカル通知で記録を促す。時間はユーザーがスロットごとに設定可能。

- 例：朝のスロット → 8:00に通知
- 例：夜のスロット → 21:00に通知
- 通知文例：「今の気分を記録しましょう 🌊」

**通知許可フロー：**
- 初回起動時には通知許可を求めない
- ユーザーが初めてリマインダーをONにした時点で、OS標準の通知許可ダイアログを表示
- Android 13+：POST_NOTIFICATIONS権限を明示的にリクエスト
- iOS：UNUserNotificationCenter.requestAuthorization() を呼び出し

### 3.7 収益化（広告・課金）

**広告：**
- ホーム画面下部にバナー広告を表示（google_mobile_ads）
- 記録入力ボトムシート内には広告を表示しない（UX優先）
- 統計画面にもバナー広告を表示

**広告除去課金：**
- 買い切り型（300〜500円）
- 購入後、全画面の広告が非表示になる
- StoreKit 2（iOS）/ Google Play Billing Library（Android）で実装
- 購入状態はローカルに保持 + レシート検証

### 3.8 日付境界

- 午前4:00を日付の切り替え点とする
- 例：3月10日 深夜2:00の記録 → 3月9日分として扱う
- 例：3月10日 午前5:00の記録 → 3月10日分として扱う

## 4. 画面設計

### 4.1 画面一覧

| No. | 画面名 | 概要 | 主な要素 |
|-----|--------|------|----------|
| 1 | ホーム（デイリー） | 今日の気分記録・波形グラフ | 波形グラフ、スロットカード、記録ボタン、バナー広告 |
| 2 | 記録入力 | 気分・メモ・タグ入力 | 気分スライダー、メモ入力、タグ選択 |
| 3 | カレンダー | 月単位の波形ミニグラフ一覧 | カレンダーグリッド、ミニ波形アイコン、連続記録バッジ |
| 4 | 統計 | 月別の傾向分析 | 時間帯別グラフ、ハイライト、バナー広告 |
| 5 | 設定 | スロット・通知・テーマ・課金 | スロット編集、通知設定、広告除去購入 |

### 4.2 画面詳細設計

#### Screen 1: ホーム（デイリービュー）

アプリ起動時のメイン画面。今日の気分の波が一目でわかる。

| エリア | 内容 |
|--------|------|
| ヘッダー | 日付（「3月10日 火曜日」）、左右矢印で日付移動 |
| 波形グラフ | 画面上部40%。なめらかな曲線で気分の浮き沈みを描画（描画アニメーション付き）。未記録スロットは点線で表示 |
| スロットカード | 横スクロールで各スロットを表示。各カード：スロット名、絵文字、メモプレビュー、タグ。未記録スロットは「記録する」ボタンを表示。長押しで編集/削除メニュー |
| バナー広告 | 画面下部（ボトムナビの上）にバナー広告を表示。広告除去済みの場合は非表示 |
| ボトムナビ | ホーム / カレンダー / 統計 / 設定（4タブ） |

**空状態（初回起動時）：**
- 波形グラフエリアにイラスト + 「最初の気分を記録してみましょう 🌊」テキスト
- 中央に大きな「記録する」ボタンを表示
- スロットカードはデフォルト3スロット（朝・昼・夜）を未記録状態で表示

#### Screen 2: 記録入力（ボトムシート）

スロットカードをタップするとボトムからスライドアップ。

| 要素 | 詳細 |
|------|------|
| 気分選択 | 5つの絵文字ボタンを横並び。選択時にバウンスアニメーション |
| メモ入力 | テキストフィールド（50文字以内）、プレースホルダー例：「会議が長かった」 |
| タグ選択 | チップ形式で複数選択可。プリセット6種 + カスタムタグ追加可能 |
| 保存ボタン | 「保存」ボタンでローカルDBに保存。保存完了時にパーティクルアニメーション。ボトムシートが閉じる |

※ 記録入力ボトムシート内には広告を表示しない（記録体験を阻害しないため）

#### Screen 3: カレンダー

月単位のカレンダーで、各日にミニ波形を表示。

- 各日のセルにミニ波形グラフ（CustomPainterで描画）
- タップするとその日のデイリービューに遷移
- 上部に「連続〇日記録中」バッジ
- 左右スワイプで月移動

**空状態：**
- 記録がない月は「まだ記録がありません」テキストを表示

#### Screen 4: 統計

月別の傾向分析画面。

- 時間帯別平均気分の棒グラフ
- 月全体の気分推移折れ線グラフ
- 「今月のハイライト」カード（最高/最低の日とメモ）
- 画面下部にバナー広告

**空状態：**
- データが不足している場合は「3日以上記録すると統計が表示されます」テキスト

#### Screen 5: 設定

- スロット管理：追加・削除・並び替え・名前変更
- 通知設定：スロットごとに通知ON/OFFと時刻設定
- テーマ切替：ライト / ダーク / システム追従
- 広告除去：購入ボタン（購入済みの場合は「広告除去済み ✓」表示）
- バージョン情報 / プライバシーポリシー

## 5. 技術仕様

### 5.1 技術スタック

| レイヤー | 技術 |
|----------|------|
| フレームワーク | Flutter 3.38+（Dart） |
| 状態管理 | Riverpod |

| ローカルDB | sqflite（SQLite） |
| 設定保存 | SharedPreferences（テーマ・広告除去状態等） |
| グラフ描画 | fl_chart（デイリー波形）+ CustomPainter（カレンダーミニ波形） |
| ローカル通知 | flutter_local_notifications |

| 課金 | in_app_purchase |
| データモデル | freezed + json_serializable |
| アニメーション | Flutter標準（AnimationController, AnimatedBuilder等） |

### 5.2 データモデル

#### mood_records テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | INTEGER PK | オートインクリメント |
| date | TEXT | 日付（yyyy-MM-dd）※午前4時で切り替え |
| slot_id | TEXT | スロット識別子（morning / afternoon / evening） |
| mood_level | INTEGER | 気分レベル（1-5） |
| memo | TEXT | ひとことメモ（50文字以内） |
| tags | TEXT | タグ（JSON配列） |
| created_at | TEXT | ISO 8601 タイムスタンプ |
| updated_at | TEXT | ISO 8601 タイムスタンプ（編集時に更新） |

#### slots テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | TEXT PK | スロットID |
| name | TEXT | 表示名（朝 / 昼 / 夜 等） |
| order_index | INTEGER | 表示順序 |
| notify_enabled | INTEGER | 通知ON/OFF（0/1） |
| notify_time | TEXT | 通知時刻（HH:mm） |
| is_deleted | INTEGER | 論理削除フラグ（0/1）。削除後も過去の記録を参照可能にする |

### 5.3 DBマイグレーション戦略

- sqfliteの `onUpgrade` コールバックでバージョン管理
- v1.0 → v1.1 のスキーマ変更に備え、初期バージョンを `version: 1` で作成
- マイグレーションスクリプトは `services/database_service.dart` 内に集約
- 破壊的変更は避け、ALTER TABLE ADD COLUMN で対応

## 8. 収益化・マーケティング

### 8.1 収益モデル

- 基本無料（バナー広告表示）
- 広告除去：買い切り課金（300〜500円）
  - 競合のサブスクモデルへの不満を逆手に取り、「買い切りで広告も消える」を訴求
  - 広告が目に入る → 「消したい」という自然な課金動機

### 8.2 広告配置ルール

| 画面 | 広告 | 理由 |
|------|------|------|
| ホーム | バナー（下部） | メイン画面で表示回数最多 |
| 記録入力（ボトムシート） | なし | 記録体験を阻害しない |
| カレンダー | なし | ミニ波形の視認性を優先 |
| 統計 | バナー（下部） | 閲覧時間が長い画面 |
| 設定 | なし | 低頻度アクセス画面 |

### 8.3 App Store最適化（ASO）

キーワード戦略：

- 主要KW：気分記録、気分日記、ムードトラッカー
- 副次KW：波形グラフ、浮き沈み、メンタルヘルス
- スクリーンショット：波形グラフが映えるスクショを最初に配置
- 「広告除去は買い切りのみ、サブスクなし」をストア説明に明記

## 9. App Store審査対策

- 完全ローカル動作のため、プライバシー関連のリジェクトリスクが低い
- ヘルスケアカテゴリに該当しないため、医療機器審査を回避
- 「気分記録」であり「診断・治療」ではないことを明確に記載（ディスクレーマー追加）
- プライバシーポリシーを用意（データ収集なしの旨を明記）
- 広告SDKのデータ収集についてApp Tracking Transparencyを考慮
